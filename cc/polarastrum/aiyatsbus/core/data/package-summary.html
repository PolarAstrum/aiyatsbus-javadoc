    AlternativeData附魔额外数据类包含附魔的可选配置信息，这些数据一般不需要填写，使用默认值即可。 定义了附魔的各种行为属性，如是否可移除、权重、是否宝藏附魔等。
    Builder
    BasicData附魔基本数据类包含附魔的基础配置信息，这些数据是附魔系统必需的。 定义了附魔的启用状态、禁用世界、标识符、名称和最大等级等基本属性。
    Dependencies依赖项管理类管理组件的依赖条件，包括 Minecraft 版本支持、数据包依赖和插件依赖。 提供依赖检查功能，确保组件在满足所有条件时才能正常使用。
    Builder
    Displayer附魔显示类负责管理附魔的显示格式和内容，包括名称、描述、等级等显示元素。 支持变量替换、占位符处理和颜色格式化。
    Limitations附魔限制管理类负责管理附魔的各种限制条件，包括冲突检查、依赖检查、权限检查等。 支持多种检查类型：获取、商人交易、铁砧合成、使用等。 通过配置文件定义限制规则，支持复杂的条件判断。
    Failed失败结果
    CheckResult检查结果密封类表示限制检查的结果，包含成功或失败状态以及失败原因。
    PlayerDataAiyatsbus 玩家数据类存储和管理玩家的附魔相关数据，包括菜单模式、收藏夹、过滤器和冷却时间。 支持数据的序列化和反序列化，用于持久化存储。
    RegistryItem注册项基类所有可注册项目的基类，包含配置节点和唯一标识符。 用于统一管理各种配置项的加载和存储。 提供基本的配置访问功能。
    Registry注册器基类提供统一的注册项管理功能，支持配置文件的自动加载和重载。 实现了 Map 接口，可以直接通过键值对的方式访问注册项。 支持依赖检查和生命周期管理。
    Variables附魔变量管理类负责管理附魔系统中的各种变量，包括变量的解析、计算和存储。 支持四种类型的变量：等级相关变量、物品相关变量、常量变量和自定义变量。 提供变量计算、修改和批量处理功能。

    Enum Description
    CheckType检查类型枚举定义了不同的检查场景和对应的限制类型。 每种检查类型都有其特定的限制类型集合。
    LimitType限制类型枚举定义了各种限制条件的类型。 每种类型都有其特定的检查逻辑和处理方式。
    MenuMode菜单模式枚举
    VariableType附魔变量类型枚举定义了四种不同类型的变量：

    • leveled：与等级有关的变量，例如等级越高触发概率越大，公式需要带入等级计算，可嵌套其他变量但严禁互相嵌套

    • modifiable：与物品强相关的数据，需要写在物品的 PDC 里，如武器击杀次数累积多少触发什么东西

    • ordinary：常量，也可以理解为配置项，不提供任何计算

    • custom：开发者自定义的变量，通过函数计算得出结果
